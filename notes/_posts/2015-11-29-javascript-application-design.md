# Introduction to Build First

- 通过重构不断提高代码质量,为代码编写测试用例, 将遗留代码放到单独接口中方便后续重构

## 1.2 Build First核心原则

- 通过减少人工操作减少错误发生可能
- 自动化常用任务增加产量
- 提高程序设计的模块化, 可扩展性
- 通过降低复杂性提高可测试性和可维护性
- 确保发布版本遵守性能最佳实践
- 代码发布之前都经过测试
